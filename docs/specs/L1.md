# L1 - High-Level Requirements

## L1-001: User account management
The system must let a user register, sign in, sign out, reset a forgotten password, manage profile information, and delete their account.

## L1-002: Equipment-typed workout session logging
The system must let a user record a workout session against one of the supported pieces of home equipment (treadmill, indoor bike, bench press, elliptical) and review, edit, duplicate, or delete past sessions.

## L1-003: Daily fitness targets and progress
The system must track daily active calories burned against a configurable target (default 1,500 kcal) and total minutes worked out, and surface progress to the user on the authenticated landing page.

## L1-004: Monthly weight-loss goal
The system must let a user set a monthly weight-loss goal (default 20 lb / month), record current weight, and show progress against the goal.

## L1-005: Behavioral guardrails (morning workout / night-eating windows)
The system must let a user configure a morning workout window and a kitchen-closed (night-eating) window, and use those windows to score behavior and trigger reminders.

## L1-006: Gamification — points, streaks, tier, rewards
The system must award points for behaviors that advance the user's goals (logging workouts, hitting morning bonus, keeping streaks, respecting the kitchen-closed window), maintain a streak and tier, and let the user redeem points against a rewards catalog.

## L1-007: Apple Watch / HealthKit integration
The system must integrate with Apple Watch via HealthKit to import active calories, heart rate, and workout sessions, and reconcile imports with manually logged sessions.

## L1-008: Notifications and reminders
The system must deliver a morning workout reminder, a kitchen-closes nudge, and configurable in-app alerts (e.g., friend leaderboard).

## L1-009: Empty, loading, and error states
The system must present clear empty, loading (skeleton), and error states for every primary surface so the user always knows what is happening and what to do next.

## L1-010: Responsive web UI
The system must function correctly and read as mobile-first across XS (<576px), SM (≥576px), MD (≥768px), LG (≥992px), and XL (≥1200px) viewports.

## L1-011: Authentication and session security
The system must authenticate users with username + password against locally stored credentials, issue signed JWT access tokens (and refresh tokens where appropriate), and reject invalid or expired tokens. PKCE and external identity providers are out of scope.

## L1-012: Authorization and RBAC
The system must enforce role-based access control end-to-end, from the database through the API to the frontend, with at minimum a `User` role and an `Admin` role.

## L1-013: Input validation
The system must validate every command, query, and request DTO server-side using FluentValidation, mapping validation failures to HTTP 400 with a `ValidationProblemDetails` body.

## L1-014: Performance and scalability
The system must meet measurable response-time and throughput targets for all primary user-facing endpoints under expected load.

## L1-015: Observability and operations
The system must emit structured logs and expose operational endpoints (e.g., health checks) sufficient to diagnose failures in production.

## L1-016: Accessibility
The system must meet WCAG 2.1 AA on every authenticated and unauthenticated surface, including touch-target size, keyboard navigation, focus management, and color contrast.

## L1-017: Data protection and privacy
The system must protect personally identifiable information at rest and in transit, never log secrets or plaintext passwords, and remove user data on account deletion.
