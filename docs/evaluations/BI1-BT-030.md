# BT-030 — List leaderboard

## Acceptance test
`backend/tests/Forge.Acceptance/Leaderboard/ListLeaderboardAcceptanceTest.cs` — one scenario:
- Two users registered. Default `LeaderboardOptIn = false`. User A seeded with 500 points, B with 300.
- B's `GET /api/leaderboard` shows B (always self) but not A.
- A toggles opt-in via `PUT /api/profile/leaderboard-opt-in`. B's next call now shows A ranked above B (more points).
- A toggles opt-in back off. B's next call hides A again. A's own call still includes themselves.

## Implementation
- `LeaderboardEntryDto(Guid UserId, string FirstName, string LastName, int Points, int Rank)`
- `ListLeaderboardQuery(int Page, int PageSize) : IRequest<IReadOnlyList<LeaderboardEntryDto>>`
- Handler depends on `IAppDbContext` + `ICurrentUser`. Loads `(User, sumLedger)` rows in a single projection scoped to `!IsDeleted && (LeaderboardOptIn || Id == callerId)`. Orders desc by points (tiebreak: last name, first name) and assigns `Rank = index + 1`.
- Pagination: defaults page=1 pageSize=25, clamped to [1, 100].
- `[Authorize] [HttpGet]` on `LeaderboardController`.
