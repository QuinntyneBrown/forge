# BT-022 — `AddPointsAndRewards` migration + base scoring + `IPointsScorer`

## Acceptance test
`backend/tests/Forge.Acceptance/Gamification/PointsScorerAcceptanceTest.cs` — two scenarios:
1. The migration seeds ≥5 `RewardCatalogItem` rows including a smoothie entry.
2. POST a 22-minute treadmill session → exactly one `PointsLedger` row appears with `Points = 44`, `Reason = Base`, and `SessionId` set to the new session.

## Implementation
- **Domain:** `PointsLedger`, `PointsLedgerReason` enum (`Base`, `MorningBonus`, `StreakMultiplier`, `Refund`, `Redemption`), `RewardCatalogItem`, `RewardRedemption`
- **DbContext:** three new `DbSet`s on `IAppDbContext` + `AppDbContext`. Configurations:
  - `PointsLedger`: `Reason` → int conversion, `Description` max 128, indexes `(UserId, CreatedAt)` and `SessionId`
  - `RewardCatalogItem`: `Name` max 80, `Description` max 400, index on `SortOrder`, `HasData` seeds 7 rows with deterministic GUIDs
  - `RewardRedemption`: index `(UserId, RedeemedAt)`
- **Migration:** `AddPointsAndRewards` (auto-generated). Applies on every API boot via `MigrateAsync()`.
- **Constants:** `Forge.Application/Gamification/ScoringConstants.cs` — `BasePointsPerMinute = 2`, `MorningBonusPoints = 25`, streak step + cap (1.50), tier thresholds (Iron → Platinum, 0..25000)
- **Abstraction:** `IPointsScorer.Score(WorkoutSession, ct)` in `Forge.Application/Abstractions`
- **Implementation:** `Forge.Infrastructure/PointsScorer.cs` — appends a Base ledger row of `BasePointsPerMinute × DurationMinutes`, `CreatedAt = _clock.UtcNow`, and saves
- **Wiring:** `CreateSessionCommandHandler` injects `IPointsScorer` + `IClock`; calls `_scorer.Score(session, ct)` after the session save returns
- **DI:** `services.AddScoped<IPointsScorer, PointsScorer>()` in `AddInfrastructure`
