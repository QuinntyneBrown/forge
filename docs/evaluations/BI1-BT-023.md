# BT-023 — Morning bonus

## Acceptance test
`backend/tests/Forge.Acceptance/Gamification/MorningBonusAcceptanceTest.cs` — two scenarios:
1. A 22-minute session started 05:12 in `America/New_York` (default window 05:00-07:30) writes both `+44 Base` and `+25 MorningBonus` ledger rows.
2. A 22-minute session started 09:00 in `America/New_York` writes only the `+44 Base` row.

## Implementation
- `PointsScorer.Score` now loads the user (`AsNoTracking`) and converts the session's `StartedAt` into the user's `TimeZoneId` via `TimeZoneInfo.ConvertTime`. If the resulting local `TimeOnly` falls within `[MorningWindowStart, MorningWindowEnd]` inclusive, a second ledger row is appended with `Reason = MorningBonus`, `Points = ScoringConstants.MorningBonusPoints`, `Description = "Morning bonus"`.
- Both Base and MorningBonus rows share the same `CreatedAt = _clock.UtcNow` so they tie together temporally.
- `TimeZoneNotFoundException` / `InvalidTimeZoneException` short-circuit to "no bonus" — a misconfigured zone never blocks scoring.

## Note
The BT-022 acceptance test was tightened to filter ledger rows by `Reason == Base` because the original assertion (a single ledger row total) became incidentally false once the morning bonus shipped — `DateTimeOffset.UtcNow` at the test runtime fell inside the default NY morning window. Filtering by reason keeps the BT-022 assertion focused on its actual contract (Base row appears with the right session attribution) without coupling it to the BT-023 behavior.
