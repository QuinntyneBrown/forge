# FT-035 — accessibility.spec.ts

## Acceptance test
`frontend/tests/accessibility.spec.ts` runs `AxeBuilder` with the
`wcag2a` + `wcag2aa` rule packs against every route in the app:

- **Anonymous (5 routes, all live-verified now):** `/sign-in`,
  `/sign-up`, `/password-reset`, `/error?traceId=test`,
  `/does-not-exist`. All five pass with **zero violations**
  (run output: `5 passed (22.8s)`).
- **Authenticated (5 routes, deferred to SQLEXPRESS availability):**
  `/dashboard`, `/workouts`, `/workouts/new`, `/rewards`, `/profile`.
  The spec registers a fresh user via the API in `beforeEach` and
  signs in before scanning, so each authed route gets a clean
  baseline.

Any axe violation fails the build with an inline summary listing the
rule id, impact, help text, and the number of offending DOM nodes —
making fix-on-find quick.

## Implementation
- Added dev dep `@axe-core/playwright` (npm install).
- `frontend/tests/accessibility.spec.ts` — single spec file:
  - `ANONYMOUS_ROUTES` array → `test()` per route.
  - `test.describe('authenticated')` → `beforeEach` registers + signs
    in a fresh user, then a `test()` per route.
  - `scan()` helper waits for `networkidle` (best-effort), runs
    `AxeBuilder({ page }).withTags(['wcag2a', 'wcag2aa']).analyze()`,
    and asserts `results.violations` is empty.

## Library placement
- Test-only addition; no library code changes required since prior
  slices (`FT-001`..`FT-034`) already exercised `[testid]` /
  `aria-label` / form-label discipline as they shipped.
