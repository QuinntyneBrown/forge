# BT-028 — Get current tier

## Acceptance test
`backend/tests/Forge.Acceptance/Rewards/GetCurrentTierAcceptanceTest.cs` — one scenario:
- Seed +4999 lifetime points → GET /api/tier → `Silver`.
- Post a 1-min mid-day session (NY local 14:00 to skip morning bonus) → lifetime ≥ 5001 → GET /api/tier → `Forged Iron`.

## Implementation
- `TierDto(string Name, int LifetimePoints, string? NextTierName, int? PointsToNextTier)`
- `GetCurrentTierQuery() : IRequest<TierDto>`
- `GetCurrentTierQueryHandler` (`IAppDbContext` + `ICurrentUser`):
  - Sums positive `PointsLedger` entries (`Points > 0`) for the user → `lifetimePoints`. Refunds/Redemptions are negative and so are intentionally excluded — redemptions don't demote you.
  - Walks `ScoringConstants.TierThresholds` ascending, picking the highest threshold whose `MinLifetimePoints ≤ lifetimePoints`.
  - Computes `NextTierName` + `PointsToNextTier` when not already at the top tier.
- `[HttpGet("/api/tier")]` on `RewardsController` (absolute route to honour the spec).
