# BT-018 — Update morning window

## Acceptance test
`backend/tests/Forge.Acceptance/Profile/UpdateMorningWindowAcceptanceTest.cs` — three scenarios:
1. PUT 05:00–08:00 + reminder on → 204; GET /api/me reflects all three fields.
2. PUT 06:30–09:00 + reminder off → 204; GET /api/me shows `morningReminderEnabled: false`.
3. PUT with start == end → 400 (FluentValidation `Start.LessThan(End)`).

## Implementation
- `UpdateMorningWindowCommand(TimeOnly Start, TimeOnly End, bool ReminderEnabled) : IRequest`
- Validator: `RuleFor(x => x.Start).LessThan(x => x.End)` — `TimeOnly` is value-typed and inherently in `[00:00, 23:59:59.9999999]`, so no separate range rule is needed
- Handler: depends on `IAppDbContext` + `ICurrentUser`; loads the user, mutates `MorningWindowStart`, `MorningWindowEnd`, `MorningReminderEnabled`, saves
- `[Authorize] [HttpPut("morning-window")]` on `ProfileController` returning `204 NoContent`
