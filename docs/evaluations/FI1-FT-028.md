# FT-028 — WorkoutDetailFormComponent + WorkoutNewPage

## Acceptance test
`frontend/tests/workout-create.spec.ts` registers a fresh user, signs in,
navigates to `/workouts/new`, fills the form (treadmill, 22 min, 2.1 mi,
128 bpm, 218 kcal), submits, and asserts the dashboard's daily-ring
value contains "218" after the redirect. Same SQLEXPRESS-availability
caveat as FT-027 — the spec is in tree; the local DB instance was
recently switched to SQLEXPRESS and that engine isn't installed on this
loop's machine.

## Implementation
### `domain` library
- `workout-detail-form/workout-detail-form.component.{ts,html,scss}`.
- Reactive form built with `FormBuilder.nonNullable.group(...)`. Validators mirror the L2-007 / BT-007 ranges:
  - `equipment` required, default `Treadmill`.
  - `date`, `time` required (HTML `type="date"` / `type="time"`).
  - `durationMinutes` 1..480.
  - `distanceMiles` ≥ 0 (hidden when equipment is `BenchPress`).
  - `avgHeartRateBpm` 30..240.
  - `activeCalories` 0..5000.
  - `notes` ≤ 2000.
- `equipmentValue` signal mirrors the equipment control so a `hideDistance` computed can drop the distance field for bench press without touching the form value. The control's value remains `null` when hidden.
- On submit the form composes `startedAt` ISO from the date + time controls, then calls `ISessionsService.create`. Emits a `(created)` event with the new session id.
- Composes `<forge-card>` + `<forge-button>` + Material form fields (input / select / textarea) per the existing form pattern.
- Public-API export added.

### `app`
- `pages/workout-new/workout-new.page.{ts,html,scss}`.
- Wraps `<forge-workout-detail-form>` in `<forge-app-shell>` with the standard four-destination nav rail; on `(created)` navigates to `/dashboard`.
- `app.routes.ts` registers `/workouts/new` behind `authGuard`.

## Library placement
- Form in `domain` (consumes `api`). ✅
- Page in `app`.
