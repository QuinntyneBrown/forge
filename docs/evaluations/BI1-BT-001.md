### Non-blocking observations

- The earlier MB1 / MB2 work created a `Forge` database via `EnsureCreated`. EF cannot migrate an `EnsureCreated`-managed schema (no `__EFMigrationsHistory` row). The transition required dropping the legacy `Forge` database before the first `MigrateAsync` boot. Documented in the runbook so a fresh dev environment doesn't hit it.
- The acceptance test creates and drops a per-test SqlExpress. On developer machines this is fast (~1 s) but on CI a shared SqlExpress instance might race; future BI1 slices that add more acceptance tests should run them sequentially via `xunit.runner.json` `parallelizeTestCollections: false` if races appear.
- The migration baseline name (`Initial_AuthAndSessions`) intentionally signals the slice scope — subsequent migrations (`AddProfileAndWeight` from BT-015, `AddPointsAndRewards` from BT-022) follow the same naming.
