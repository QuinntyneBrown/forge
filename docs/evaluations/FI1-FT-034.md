# FT-034 — NotFoundPage + empty-state spec

## Acceptance test
`frontend/tests/empty-state.spec.ts` — two scenarios:
1. Register a fresh user, sign in, navigate to `/workouts`. Assert the `workout-list-empty` panel renders with "No sessions yet" + "Log your first session" CTA. (Pending SQLEXPRESS for live execution.)
2. Anonymous browser visits `/does-not-exist`; assert `NotFoundPage` renders with "Page not found" copy and a `not-found-home` CTA. (Runnable without backend.)

## Implementation
- `pages/not-found/not-found.page.{ts,html,scss}` — composes `<forge-empty-state>` + `<forge-button>`. CTA navigates to `/dashboard` (the auth guard will redirect anonymous users to `/sign-in`, which is the right behaviour since the user landed on a 404 in an unknown auth state).
- `app.routes.ts` registers `{ path: '**', component: NotFoundPage }` as the last route so any unmatched path resolves to the 404 page.

## Library placement
- Page in `app`. Reuses `EmptyStateComponent` + `ButtonComponent` from the `components` library — no new artifacts in `domain`. ✅
