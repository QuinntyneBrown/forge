# BT-016 — Record current weight

## Acceptance test
`backend/tests/Forge.Acceptance/Profile/RecordCurrentWeightAcceptanceTest.cs` — two scenarios:
1. Two consecutive POSTs (195.4, 194.8) both persist; `WeightEntries` contains two rows ordered by `RecordedAt`.
2. POST `{ weightLb: 0 }` returns 400 ProblemDetails (FluentValidation `GreaterThan(0)`).

## Implementation
- `WeightEntry { Id, UserId, WeightLb (decimal(6,2)), RecordedAt }` domain class
- `IAppDbContext.WeightEntries` DbSet; `OnModelCreating` PK + decimal precision + composite index `(UserId, RecordedAt)`
- `AddWeightEntries` EF migration applies on boot via `MigrateAsync`
- `RecordCurrentWeightCommand(decimal WeightLb) : IRequest`
- Validator: `GreaterThan(0).LessThanOrEqualTo(1500)`
- Handler: depends on `IAppDbContext` + `ICurrentUser` + `IClock`; appends a row stamped `RecordedAt = _clock.UtcNow`; throws `UnauthorizedAccessException` if no user
- `[Authorize] [HttpPost("api/profile/weight")] → 204` on `ProfileController`
