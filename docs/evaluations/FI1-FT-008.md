# FT-008 — ProgressRingComponent

## Acceptance test
Per the FT-008 entry, no standalone Playwright spec is required — verified inside `dashboard.spec.ts` (FT-024) and `rewards-redeem.spec.ts` (FT-030) when those land.

## Implementation
- `frontend/projects/components/src/lib/progress-ring/progress-ring.component.{ts,html,scss}`.
- Inputs: `value`, `max` (defaults 0/100), plus optional `diameter`, `strokeWidth`, `testid`.
- A `percentage` getter computes `value/max*100`, clamped to `[0, 100]` and guarded against `max <= 0` so the spinner never receives `NaN`.
- The template wraps `<mat-progress-spinner mode="determinate">` in a relatively-positioned container with an absolute-positioned `<ng-content>` slot — consumers can drop centered text (e.g. `980 / 1500 kcal`) inside the ring.
- Public-API export added.

## Library placement
- `components` library, no imports from `api` / `domain`. ✅
