# FT-023 — LeaderboardCardComponent + LeaderboardService

## Implementation

- `models/leaderboard-entry.model.ts` — `LeaderboardEntry { userId, firstName, lastName, points, rank }` matching the BT-030 backend payload.
- `leaderboard.service.contract.ts` — `ILeaderboardService.list(page?, pageSize?)` + `LEADERBOARD_SERVICE` token.
- `leaderboard.service.ts` — calls `GET ${API_BASE_URL}/api/leaderboard?page=&pageSize=` with `HttpParams`.
- Public-API exports added.

### `domain` library
- `leaderboard-card/leaderboard-card.component.{ts,html,scss}`.
- Loads top 5 entries (`leaderboard.list(1, 5)`); state held in signals.
- Empty state copy: "No one to compare with yet." when the list is empty (e.g. only the caller, who's opted out by default).
- Renders rank / name / points in a 3-column grid with tabular-nums on the points column for clean alignment.
- BEM class names + `data-testid` attributes for FT-024.
- Public-API export added.

### `app` config
- `{ provide: LEADERBOARD_SERVICE, useClass: LeaderboardService }` registered.
