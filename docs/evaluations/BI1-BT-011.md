# BT-011 — Duplicate session

## Acceptance test
`backend/tests/Forge.Acceptance/Sessions/DuplicateSessionAcceptanceTest.cs` — one scenario:
- Create a 5-day-old 22-min treadmill session at NY 14:00. POST `/api/sessions/{id}/duplicate`. The new session has the same equipment, duration, distance, HR, calories, notes — but `StartedAt` is "now" (well after the original). A fresh `+44 Base` ledger row is attached to the new session id.

## Implementation
- `DuplicateSessionCommand(Guid Id) : IRequest<DuplicateSessionResult>` + validator (`Id.NotEmpty()`)
- `DuplicateSessionResult(Guid Id)`
- Handler (`IAppDbContext` + `ICurrentUser` + `IClock` + `IPointsScorer`):
  - Loads source session scoped to caller (`AsNoTracking`); throws `SessionNotFoundException` if missing.
  - Inserts a copy with a fresh `Guid`, `StartedAt = _clock.UtcNow`, `CreatedAt = _clock.UtcNow`, all other fields copied verbatim.
  - SaveChanges, then `_scorer.Score(copy)` so the duplicate gets its own Base / MorningBonus / StreakMultiplier rows against the new state.
- `[HttpPost("{id:guid}/duplicate")]` on `SessionsController` returns `201 Created` pointing at `GetById`.
