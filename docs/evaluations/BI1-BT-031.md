# BT-031 — HealthKit ingest stub

## Acceptance test
`backend/tests/Forge.Acceptance/HealthKit/HealthKitIngestAcceptanceTest.cs` — one scenario:
- POST `{ sampleType: "ActiveEnergyBurned", value: 245.5, unit: "kcal", recordedAt }` → response `202 Accepted`. A captured ILoggerProvider sees a structured log line containing `healthkit.ingest.deferred`.

## Implementation
- **Abstraction:** `IHealthKitIngest.IngestAsync(userId, sampleType, value, unit, recordedAt, ct)` in `Forge.Application/Abstractions`.
- **Deferred no-op:** `Forge.Infrastructure/Deferred/LoggingHealthKitIngest.cs` logs `healthkit.ingest.deferred` with all fields. Doc-commented as a deferred integration per BP1 §8.
- **Command + validator + handler:** `IngestHealthKitSampleCommand(SampleType, Value, Unit, RecordedAt) : IRequest`. Validator: `SampleType` non-empty ≤64, `Unit` non-empty ≤16, `Value ≥ 0`. Handler: pulls `userId` from `ICurrentUser`, delegates to `IHealthKitIngest`.
- **Controller:** `[Authorize] [HttpPost("/api/healthkit/ingest")]` returns `Accepted()` (202).
- **DI:** `services.AddScoped<IHealthKitIngest, LoggingHealthKitIngest>()` alongside the existing email-sender registration.
