# BT-009 — List sessions with filters

## Acceptance test
`backend/tests/Forge.Acceptance/Sessions/ListSessionsAcceptanceTest.cs` — four scenarios:
1. `?equipment=Treadmill` returns only treadmill rows.
2. `?range=week` returns only sessions whose `StartedAt` is within 7 days of `IClock.UtcNow`.
3. `?search=zone` returns only rows whose `Notes` LIKE-match "zone" (case-insensitive via SQL collation).
4. A second user listing with `?range=all` sees zero rows (per-user scoping).

## Implementation
- `SessionRange` enum (All, Today, Week, Month) — public so the controller can bind `?range=`
- `ListSessionsQuery(EquipmentType?, SessionRange, string?, int, int) : IRequest<SessionPage>`
- `SessionPage(IReadOnlyList<SessionDto>, int Page, int PageSize, int Total)`
- `ListSessionsQueryHandler` depends on `IAppDbContext` + `ICurrentUser` + `IClock`; uses `.AsNoTracking()`, scopes to `userId`, applies filters, projects to `SessionDto`, paginates
- Range thresholds derived from `_clock.UtcNow`: Today → start of UTC day; Week → -7d; Month → -30d
- Search uses `EF.Functions.Like("%term%")` so the SQL Server LIKE plan can use the `Notes` column (case-insensitivity comes from the default collation `SQL_Latin1_General_CP1_CI_AS`)
- Page defaulted to 1, pageSize defaulted to 50, clamped to [1, 200]
- Composite index `WorkoutSessions(UserId, StartedAt)` from BT-001 supports `OrderByDescending(s => s.StartedAt)` after the user-scope filter
- `[Authorize] [HttpGet]` action returns `Ok(SessionPage)`
