### Implementation

`BadgeComponent` (in `components/`):
- Wraps `<mat-chip class="badge" [class.badge--<variant>]="variant === '<variant>'" [attr.data-testid]="testid" disableRipple>`.
- Inputs: `[variant: 'success' | 'warning' | 'error' | 'neutral']` defaulting to `'neutral'`, `[testid]`. Same `testid ‚Üí [attr.data-testid]` pattern from FT-002 so Playwright's `getByTestId` resolves to the inner element.
- `disableRipple` is set because status pills aren't interactive ‚Äî eliminates Material's tap ripple overlay that would otherwise activate on click.
- SCSS maps each variant to Material's chip CSS custom properties (`--mdc-chip-elevated-container-color` + `--mdc-chip-label-text-color`) driven by the M3 design tokens already wired in `styles.scss`.

`HealthBadgeComponent` (in `domain/`):
- Now composes `<forge-badge [variant]="badgeVariant()" testid="health-badge">` instead of the previous hand-rolled `<div>`. `badgeVariant` is a `computed<BadgeVariant>` derived from `isHealthy()` ‚Üí `'success'` or `'error'`.
- The dot + label remain inside the badge as projected `<ng-content>`.
- SCSS pruned to documentation stubs; the chip's container color, padding, and shape now come from Material via the variant tokens.
