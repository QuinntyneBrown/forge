# BT-012 — Delete session with point refund

## Acceptance test
`backend/tests/Forge.Acceptance/Sessions/DeleteSessionAcceptanceTest.cs` — two scenarios:
1. Create a 22-min treadmill session at NY 14:00 → balance becomes 44. DELETE the session → 204 NoContent → balance back to 0. The session row is gone, and a single Refund row of `-44` linked to the session id remains in the ledger as audit trail.
2. DELETE a non-existent session id → 404 ProblemDetails ("Session not found.").

## Implementation
- `DeleteSessionCommand(Guid Id) : IRequest` + validator (`Id.NotEmpty()`).
- Handler (`IAppDbContext` + `ICurrentUser` + `IPointsScorer`):
  - Loads session scoped to caller; throws `SessionNotFoundException` if missing.
  - Calls `_scorer.Refund(session, ct)` — appends one compensating row of `-awarded` referencing the session id.
  - Removes the session row, `SaveChangesAsync`. The Refund ledger row stays as a "tombstone" with `SessionId` pointing at the now-deleted session for audit purposes.
- `[HttpDelete("{id:guid}")]` on `SessionsController` returns `204 NoContent`.

| # | Criterion | Result |
