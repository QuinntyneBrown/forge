# FT-025 — SessionsService + contract

## Acceptance test
Per the FT-025 entry, end-to-end coverage lands with FT-027 / FT-028 / FT-029. The slice itself is plumbing.

## Implementation
### Models in `api/lib/models/`
- `session.model.ts` — `Session` (matches BT-007 SessionDto), `SessionPage` (BT-009 paginated wrapper), and shared `EquipmentType` union (`'Treadmill' | 'IndoorBike' | 'BenchPress' | 'Elliptical'`).
- `session-list-query.model.ts` — `SessionListQuery` filter shape with optional `equipment`, `range`, `search`, `page`, `pageSize`. `SessionRange = 'all' | 'today' | 'week' | 'month'` lower-cased on the wire to match the backend `[FromQuery] SessionRange` enum binding (the service capitalises the first letter when serialising).
- `create-session-request.model.ts`, `update-session-request.model.ts` — body shapes mirroring BT-007 / BT-010 controllers.

### Service in `api/`
- `sessions.service.contract.ts` — `ISessionsService` exposing `list(query?)`, `getById(id)`, `create(req)`, `update(id, req)`, `duplicate(id)`, `delete(id)`. `SESSIONS_SERVICE` injection token.
- `sessions.service.ts` — `SessionsService` impl. `list` builds an `HttpParams` from the optional fields and capitalises the range to match the backend enum. `update` returns `Observable<void>` (PUT → 204). `duplicate` is a `POST {id}/duplicate` returning `{ id }`. `delete` returns `Observable<void>` (DELETE → 204).

### `app` config
- `{ provide: SESSIONS_SERVICE, useClass: SessionsService }` registered alongside the other service bindings.
