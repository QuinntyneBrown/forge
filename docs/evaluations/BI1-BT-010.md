# BT-010 — Update session

## Acceptance test
`backend/tests/Forge.Acceptance/Sessions/UpdateSessionAcceptanceTest.cs` — two scenarios:
1. Material change: a 22-min treadmill session at NY 14:00 (no morning bonus, day-1 streak = 0 row) is updated to 30 min. Ledger reads `+44 Base`, `-44 Refund`, `+60 Base` (or compatible) — net = 60.
2. Notes-only edit: ledger remains a single `+44 Base` row; the `Notes` column reflects the new value.

## Implementation
- Extended `IPointsScorer` with `Refund(WorkoutSession session, ct)` — sums prior ledger rows for the session and appends a single compensating `Refund` row of `-awarded`. No-ops if there's nothing to refund. (Satisfies the BT-025 contract preemptively.)
- `UpdateSessionCommand(Id, Equipment, StartedAt, DurationMinutes, DistanceMiles, AvgHeartRateBpm, ActiveCalories, Notes)` + validator (mirrors `CreateSessionCommandValidator` plus `Id.NotEmpty()`).
- `UpdateSessionCommandHandler` (`IAppDbContext` + `ICurrentUser` + `IPointsScorer`):
  1. Loads session scoped to caller; throws `SessionNotFoundException` if missing.
  2. Detects "material" change = any of `Equipment`, `StartedAt`, `DurationMinutes` differs.
  3. If material → `_scorer.Refund(session, ct)` BEFORE mutating fields.
  4. Mutates all fields, `SaveChangesAsync`.
  5. If material → `_scorer.Score(session, ct)` to write fresh Base / MorningBonus / StreakMultiplier rows against the new state.
- `[HttpPut("{id:guid}")]` on `SessionsController` returns `204 NoContent`.
- `ExceptionHandlingMiddleware` maps `SessionNotFoundException` → 404 ProblemDetails.

## Note on BT-025 overlap
The slice spec for BT-010 says "deletes prior PointsLedger rows for the session and re-scores"; the BT-025 spec says the actual contract is `IPointsScorer.Refund(sessionId)` writing compensating rows. The two are mutually exclusive; the audit-friendly compensation pattern is the one in code now, satisfying both BT-010 and BT-025 simultaneously. BT-025 will get its own pass-through evaluation when the dependent BT-012 (delete) lands.
