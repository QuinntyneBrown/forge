# BT-019 — Update kitchen window + nudge toggle

## Acceptance test
`backend/tests/Forge.Acceptance/Profile/UpdateKitchenWindowAcceptanceTest.cs` — two scenarios:
1. PUT 21:00 → 05:30 (spans midnight) + nudge off → 204; GET /api/me reflects all three fields.
2. PUT 20:00 == 20:00 → 400 (FluentValidation `Start != End`).

## Implementation
- `UpdateKitchenWindowCommand(TimeOnly Start, TimeOnly End, bool NudgeEnabled) : IRequest`
- Validator: `RuleFor(x => x).Must(c => c.Start != c.End)` — span-midnight is intentional, only zero-length is rejected
- Handler: depends on `IAppDbContext` + `ICurrentUser`; loads the user, mutates `KitchenClosedStart`, `KitchenClosedEnd`, `KitchenNudgeEnabled`, saves
- `[Authorize] [HttpPut("kitchen-window")]` on `ProfileController` returning `204 NoContent`
