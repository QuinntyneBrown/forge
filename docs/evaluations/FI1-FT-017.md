### Implementation

- **`BottomNavComponent`** (`<forge-bottom-nav>`): wraps a `<nav class="bottom-nav">` containing one `<a routerLink>` per destination. Each item is a column-flex `mat-icon` + label, ≥ 56px tall (L2-046 touch target). Sticky to the bottom of the viewport via `:host { position: sticky; bottom: 0 }`. `routerLinkActive="bottom-nav__item--active"` toggles the primary-color highlight on the current route. Carries `data-testid="app-shell-bottom-nav"`.
- **`NavRailComponent`** (`<forge-nav-rail>`): wraps `<mat-nav-list>` with one `<a mat-list-item routerLink>` per destination. Sticky-positioned at `top: 0` with `height: 100vh` so the rail stays anchored as page content scrolls. Carries `data-testid="app-shell-nav-rail"`.
- **`AppShellComponent`** (`<forge-app-shell>`):
  - Imports Angular CDK's `BreakpointObserver`. A `(min-width: 992px)` query feeds a signal (`isDesktop`) which derives `showNavRail` and `showBottomNav` computeds.
  - Template: `<mat-toolbar>` brand bar at top → body row that conditionally embeds `<forge-nav-rail>` for desktop → `<main>` slot projecting `<ng-content>` → conditionally appends `<forge-bottom-nav>` for mobile.
  - Inputs: `[destinations]` (required), `[title]` (defaults to "Forge Fit"). The destinations array carries `{ label, icon, routerLink }` for the four primary destinations (Home / Workouts / Rewards / Profile).
- **`DashboardPage`** (in `forge`): pulled the bespoke `<main class="dashboard">` inside `<forge-app-shell [destinations]="destinations">`. The dashboard's `<header>` and `<section class="dashboard__grid">` continue to render as before; only the surrounding chrome changes. The destinations array is a static class field defined in the component.

For this slice, leaving the dashboard's button intact keeps the diff focused.
- The `@Input({ required: true }) destinations: NavDestination[]` declaration triggers Angular 18+'s required-input check at compile time. The default `[]` initializer is required for the class instantiation but the consumer must supply a value or the build fails — exactly what the slice wants.
- `BreakpointObserver` is the standard Angular CDK primitive for responsive logic; using it (rather than `window.matchMedia` directly or a host listener) keeps the shell SSR-safe (runtime checks happen only in the observer subscription, not at construction time).
