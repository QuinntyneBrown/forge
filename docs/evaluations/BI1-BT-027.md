# BT-027 — Redeem reward

## Acceptance test
`backend/tests/Forge.Acceptance/Rewards/RedeemRewardAcceptanceTest.cs` — two scenarios:
1. **Sufficient balance** — seed +1000 ledger, redeem the 200-pt smoothie. Response 200 with `{ redemptionId, remainingBalance: 800 }`. A `RewardRedemption` row is created. A `Redemption` ledger row of `-200` is appended with `RedemptionId` linking back to the row. Computed balance = 800.
2. **Insufficient balance** — seed +50 ledger, redeem the most expensive reward. Response 400 ProblemDetails with `title: "INSUFFICIENT_POINTS"`. No redemption rows created. Balance unchanged at 50.

## Implementation
- `RedeemRewardCommand(Guid RewardId) : IRequest<RedeemRewardResult>` + validator (`RewardId.NotEmpty()`)
- `RedeemRewardResult(Guid RedemptionId, int RemainingBalance)`
- Handler (`IAppDbContext` + `ICurrentUser` + `IClock`):
  - Loads the active catalog row by id; throws `RewardNotFoundException` if missing/inactive.
  - Sums `PointsLedger.Points` for the user; throws `InsufficientPointsException(balance, cost)` if below cost.
  - Inserts a `RewardRedemption` row plus a `-cost` `Redemption` ledger row stamped via `IClock.UtcNow`.
- `[HttpPost("{id:guid}/redeem")]` on `RewardsController` returns `Ok(result)`.
- `ExceptionHandlingMiddleware` maps:
  - `InsufficientPointsException` → 400 `application/problem+json` with `title = "INSUFFICIENT_POINTS"`
  - `RewardNotFoundException` → 404 `application/problem+json`
