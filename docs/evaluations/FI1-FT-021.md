# FT-021 — WeightProgressCardComponent

## Acceptance test
End-to-end coverage in `dashboard.spec.ts` (FT-024). FT-021 itself wires `IDashboardService` into a card.

## Implementation
- `frontend/projects/domain/src/lib/weight-progress-card/weight-progress-card.component.{ts,html,scss}`.
- Injects `IDashboardService`. State held in signals.
- Computed labels:
  - `goalLabel` → `−{monthlyWeightGoalLb} lb / month`
  - `statusLabel` → "On track" if MTD loss ≥ pro-rated goal for today within current month, else "Behind"; "No goal set" when goal ≤ 0
  - `mtdLossLabel` → `−{value.toFixed(1)} lb so far in {Month}`
- `currentMonth` resolves the local month name client-side. The pro-rating uses `dayOfMonth / daysInMonth × goal` so a user who has lost 5.2 lb on May 10 (10/31 of month) against a 20 lb / month goal compares against an expected 6.45 lb — flags as behind.
- Composes `<forge-card>` only; no other Material wrappers needed.
- BEM class names + `data-testid` attributes for FT-024 to assert.
- Public-API export added.

- `domain` library, imports `api` + `components`. ✅
